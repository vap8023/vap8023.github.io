# 文件夹示例
大致可以分为以下几类：
- 00InBox（下载目录 所有新的内容）
- 01MyDoc（待归纳的 或者不知道放哪里的）
- 02Archive（按照项目分 ）
- 03Resource（网上下载内容）
	- 安装包
- 04Devlop （开发目录）
	- Project (项目目录)
	- Repository （Maven 仓库）
- Software  （软件安装目录）
# 流程
文件夹示例与子弹笔记配合使用；
- 每次接到任务，在每日子弹笔记进行记录；如果有相关资料，就在00InBox文件夹建立对应任务文件夹，文件的命名规范直接按照项目或需求名称命名_日期等；
- 任务完成后 转移01MyDoc （笔记软件+文件夹）
- 有时间归纳就归纳到 02Archive
	- 归纳可以按照项目分类；类别分类；
	- 目录分类建议采取麦肯锡提出的MECE原则；意思 “相互独立，完全穷尽”

# 文件命名规范
项目_标签_详细文档标题_版本_日期等，方便检索
